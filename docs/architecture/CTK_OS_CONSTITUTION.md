# CallMeTK OS Constitution

## Mission
CallMeTK OS exists to increase creator ownership, trust, and creative leverage. It reduces repetitive work without replacing artistic intent, human judgment, or control.

## Core principles
1. **Creator control:** the creator remains the final authority.
2. **Explain before acting:** recommendations must include confidence, reasoning, evidence, alternatives, and approval state.
3. **Local first:** use deterministic local tools before paid or remote AI when practical.
4. **Ownership by default:** source assets, masters, private data, and credentials remain outside the public code repository.
5. **One creative memory:** applications share consistent IDs, metadata, relationships, and decision history.
6. **Preserve, then expand:** existing applications are integrated, not discarded.
7. **Modular services:** CTK Core coordinates reusable services; applications remain focused.
8. **Trust is learned:** maturity is measured by reviewed decisions and category-specific agreement rates, not asset volume alone.
9. **No false certainty:** uncertain recommendations must be flagged for review.
10. **Open architecture:** providers, models, storage backends, and applications may change without rewriting the philosophy.

## CTK test
Before a feature enters the core, ask:

1. Does it increase creator ownership?
2. Does it reduce repetitive work?
3. Can its recommendation be explained transparently?
4. Can it operate locally when practical?
5. Can more than one CTK application reuse it?

Features that fail most of these questions should remain application-specific or become optional plugins.
