## Private documents recommended later

- complete DiD source material
- company operating procedures
- internal brand strategy
- proprietary prompt library
- model evaluation results tied to private assets
- business, legal, financial, roster, or licensing records
- raw conversation exports

## Review states

Every foundational document should have one status:

- `historical-source`
- `draft`
- `approved`
- `superseded`
- `private`

Only approved, collaborator-safe documents belong in the public documentation set.
